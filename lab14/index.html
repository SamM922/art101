Past Experiences

One of the most infamous examples of issues I couldn't find with my code was for Lab 7. None of my JavaScript was working, but it turns out all of it was working perfectly well and in the wrong file! A lot of debugging involves carefully reading through your code for anything you missed or typed wrong so knowing exactly what your code should look like and using neat spacing and comments are two very useful tips.
